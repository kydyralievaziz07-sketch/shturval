# Для 1С-программиста: отдать через API данные отчёта «Рейтинг продаваемого товара»

Здравствуйте! Нужен ещё один метод в HTTP-сервисе выгрузки (`yarosDataGate`).

## Что нужно

Отдавать через API **те же данные, что в отчёте «Рейтинг продаваемого товара»** за
указанную дату (тот самый, что выгружается в Excel). Группировать и оформлять ничего не
нужно — **структурируем у себя сами**. Просто дайте данные как есть.

Главное, что нам нужно по каждому проданному товару за день:
- **код** товара,
- **название**,
- **отдел / группа**, к которой он относится (как в отчёте: 1-этаж, Детская одежда,
  Женская одежда, Мужская одежда, Обувь и т.д.) — это самое важное,
- **количество** и **сумма** (и **прибыль**, если несложно).

## Формат

Любой, какой вам удобнее — например, JSON-список строк:

```
/salesReport?date=2026-06-21
->
{
  "date": "2026-06-21",
  "rows": [
    { "code": "00000000450", "name": "Резинка для волос 60с",
      "department": "1-этаж", "qty": 20, "sum": 1200, "profit": 600 },
    { "code": "Ц0000020412", "name": "Заколка 1шт 5сом",
      "department": "1-этаж", "qty": 16, "sum": 80, "profit": 30 }
    // … все проданные за день позиции
  ]
}
```

Поле `department` — отдел/группа товара (верхний уровень из отчёта). Остальное — как в
отчёте. Желательна поддержка периода: `?from=2026-06-01&to=2026-06-21`.

## Зачем

У нас в приложении каждый отдел имеет дневной план продаж. Получая эти строки, мы сами
сгруппируем продажи по отделам и сравним с планом — без ручного ввода.

## Важно

Данные у вас уже есть — это и есть отчёт «Рейтинг продаваемого товара». Нужно лишь отдать
его содержимое через API по дате. Формат — любой удобный вам, мы подстроимся. Спасибо!
