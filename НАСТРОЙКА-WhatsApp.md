# Подключение WhatsApp к «Штурвалу» (официальный Cloud API от Meta)

Это пошаговая инструкция простыми словами. Код уже готов — осталось получить
доступы у Meta и вписать их в Render. Делается один раз.

WhatsApp здесь работает **так же, как Instagram**: клиент пишет на ваш номер →
сообщение появляется во вкладке **WhatsApp** → вы (или ИИ-бот «Акылай») отвечаете.

---

## Что вам понадобится
- **Отдельный номер телефона** для бизнеса. Важно: этот номер **не должен быть
  привязан к обычному приложению WhatsApp / WhatsApp Business**. Если на нём уже
  есть WhatsApp — сначала удалите там аккаунт, либо возьмите новый номер (можно
  виртуальный). На этот номер придёт код подтверждения по SMS или звонком.
- Аккаунт Facebook (тот же, через который делали Instagram и Рекламу — подойдёт).

---

## Шаг 1. Таблицы в базе (Supabase) — один раз
Откройте Supabase → ваш проект → **SQL Editor** → вставьте и выполните:

```sql
-- Переписки WhatsApp (зеркало ig_inbox)
create table if not exists wa_inbox (
  id           bigint generated by default as identity primary key,
  company_id   text not null,
  sender_id    text,
  recipient_id text,
  mid          text,
  text         text,
  ts           bigint,
  direction    text,
  raw          jsonb,
  created_at   timestamptz default now()
);
create index if not exists wa_inbox_company_ts on wa_inbox (company_id, ts desc);

-- Подключённые номера (если номер один — можно не заполнять, берётся из Render)
create table if not exists wa_accounts (
  company_id text not null,
  phone_id   text not null,
  display    text,
  token      text,
  unique (company_id, phone_id)
);

-- Имена клиентов (приходят от WhatsApp сами)
create table if not exists wa_names (
  company_id text not null,
  waid       text not null,
  name       text,
  unique (company_id, waid)
);
```

> Если базы Supabase нет — переписки всё равно будут приходить и отвечаться, но
> не сохранятся после перезапуска сервера. Для нормальной работы база нужна.

## Шаг 2. Приложение Meta и номер WhatsApp
1. Зайдите на **developers.facebook.com** → ваше приложение (то же, что для
   Instagram/Рекламы; или создайте новое типа «Business»).
2. В приложении найдите продукт **WhatsApp** → **Set up / Начать**.
3. В разделе **WhatsApp → API Setup** добавьте свой бизнес-номер: нажмите
   **Add phone number**, введите номер, подтвердите кодом из SMS/звонка.
4. Запомните два значения с этой же страницы:
   - **Phone number ID** (длинное число) — это `WA_PHONE_ID`.
   - **WhatsApp Business Account ID** — пригодится для подписки на webhook.

---

## Шаг 3. Постоянный токен (чтобы не протухал каждые 24 часа)
Временный токен на странице API Setup живёт сутки. Нужен постоянный:
1. **business.facebook.com** → **Настройки компании** → **Пользователи → Системные
   пользователи** → создайте системного пользователя (роль Admin).
2. Нажмите **Создать токен (Generate token)**, выберите ваше приложение и
   отметьте права: **whatsapp_business_messaging** и **whatsapp_business_management**.
3. Скопируйте токен — это `WA_TOKEN`. Покажут один раз, сохраните сразу.
4. Там же привяжите системного пользователя к вашему WhatsApp-аккаунту (Assets →
   WhatsApp accounts → дать доступ), иначе токен не сможет слать сообщения.

---

## Шаг 4. Вписать доступы в Render
Render → сервис **shturval-backend** → **Environment** → добавьте переменные:

| Переменная | Что вписать |
|---|---|
| `WA_TOKEN` | постоянный токен из Шага 3 |
| `WA_PHONE_ID` | Phone number ID из Шага 2 |
| `WA_VERIFY_TOKEN` | любое своё слово-пароль, напр. `shturval-wa-2026` (понадобится в Шаге 5) |
| `WA_APP_SECRET` | App Secret приложения Meta (Settings → Basic → App Secret). Если уже задан `IG_APP_SECRET` от того же приложения — можно не добавлять, возьмётся он |
| `WA_DISPLAY` | (необязательно) как показывать номер, напр. `+996 555 12-34-56` |

Сохраните — Render перезапустит сервер сам.

---

## Шаг 5. Подключить webhook (чтобы входящие приходили в «Штурвал»)
1. В приложении Meta → **WhatsApp → Configuration → Webhook** → **Edit**.
2. **Callback URL**: `https://<адрес-вашего-Render>/api/wa/webhook`
   (например `https://shturval-backend.onrender.com/api/wa/webhook`).
3. **Verify token**: то же слово, что вписали в `WA_VERIFY_TOKEN`.
4. Нажмите **Verify and save** — Meta проверит адрес (должно стать зелёным).
5. В списке полей (**Webhook fields**) нажмите **Subscribe** напротив **messages**.

Готово. Напишите со своего личного телефона на бизнес-номер — сообщение
появится во вкладке **🟢 WhatsApp** в «Штурвале».

---

## Важно знать про правила WhatsApp
- **Окно 24 часа.** Свободным текстом отвечать можно только в течение 24 часов
  после последнего сообщения клиента. Позже — лишь заранее одобренные Meta
  «шаблоны» (это отдельная настройка, сделаем при необходимости).
- **Первым писать нельзя** просто так — только клиентам, которые написали вам, и
  только в окне 24 ч (или шаблоном).
- **ИИ-бот** — общий с Instagram: включается во вкладке Instagram → «ИИ-бот».
  Если оператор ответил в диалоге хоть раз — бот в этом диалоге замолкает.

## Если что-то не работает
- Webhook не верифицируется → проверьте, что `WA_VERIFY_TOKEN` в Render и в Meta
  совпадают, а сервер Render «проснулся» (бесплатный план засыпает).
- Сообщения не приходят → проверьте, что подписаны на поле **messages** (Шаг 5.5).
- Не отправляется ответ → токен без нужных прав или прошло окно 24 ч.
